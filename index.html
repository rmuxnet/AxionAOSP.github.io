Frequently Asked Questions

Why is AxionAOSP a performance-oriented ROM?

AxionAOSP incorporates various performance optimizations from ProtonAOSP, along with our own efficiency tweaks to reduce system resource usage.


Some key enhancements include:

- Optimized notification icon and image memory usage – commit
- Compressed and optimized media artwork album art – commit
- Lowered snapshot memory usage – commit
- Lowered max broadcast limit – commit
- Reduced job concurrency limit – commit
- Adjusted max process limits for legacy devices – commit
- Optimized storage management for low-end devices: commit, commit


Additionally, we avoid adding features unless they are verified to have no negative impact on memory or performance.

Can you add Feature XX?

We welcome suggestions, but features will only be added if they are useful and non-intrusive.


Example of features that will not be added:
- Proprietary features (e.g., Pixel Framework)
- Resource-intensive features
- Features that may introduce memory leaks
- Spoofing of play intergrity and google apps

Is battery life good on AxionAOSP?

Battery life is subjective and depends on factors such as kernel, usage patterns, and device configuration. However, AxionAOSP includes optimizations aimed at maximizing efficiency. See Why is AxionAOSP a performance-oriented ROM? for details.

Why isn't Play Integrity passing?

Like LineageOS, we choose not to spoof Play Integrity or modify apps to pass Google's certification checks.

Where are the customizations?

AxionAOSP is not a customization-focused ROM. If extensive customization is a priority for you, other ROMs may better suit your needs.

How do I install AxionAOSP?

Follow our detailed installation guide on the documentation page. Ensure your device is supported before proceeding.
